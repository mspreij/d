This is a mix of escape sequences and stty settings.

The topleft coordinate for cursor moving is [1,1]

# some random sequences that can be used in scripts:
\\e[2J     \e[1mclear\e[0m the screen
\\e[\e[32mn\e[0m;\e[38;5;201mm\e[0mH   \e[1mmove\e[0m cursor to \e[38;5;201mm\e[0m'th column, \e[32mn\e[0m'th row (yes, Y comes first). Top-left is 1;1, but as the values default to 1
          \\e[H moves the cursor there. \\e[\e[38;5;201m5\e[0mH will move it to the 5th row (and 1st column), \\e[;\e[32m3\e[0mH to 3rd column (top row)
\\e[\e[32mn\e[0mA     \e[1mmove\e[0m cursor \e[32mn\e[0m places up
\\e[\e[32mn\e[0mB                ...down
\\e[\e[32mn\e[0mC          ...right
\\e[\e[32mn\e[0mD     and left.
...

# mouse mode
There are two sequences for mouse input, one lets you select what types of input to receive:
\\e[?1000h	Report button press/release only ("normal tracking")
\\e[?1001h	Highlight tracking (rarely supported)
\\e[?1002h	Report button press, release, and movement while a button is held (drag tracking)
\\e[?1003h	Report all mouse movement, even with no button pressed

The second sets the coordinate system. There were earlier versions for smaller screens, but
  basically, you just want SGR mouse mode:
To enable \\e[?1006h , disable \\e[?1006l
Inputs will arrive as \\e[<\e[38;5;201minput\e[0m;\e[38;5;201mx\e[m;\e[38;5;201my\e[mM
"input" here is a bitfield (am I using that right? it's late) which contains control, shift, meta,
alt, left/middle/right mouse button, scroll, or even move. So it can tell you "user alt-dragged an
item here". then x and y coords, and M/m for button down or up.
